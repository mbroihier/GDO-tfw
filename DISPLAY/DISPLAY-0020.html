Test Case DISPLAY-0020
Title Display of Authentication Failure
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the Android app is started a main page shall display as a minimum:
  • A text field that displays the status:
  • The app shall display a status of "Authentication Failure" when it fails to send a message considered valid by the controller.

The first time the controller is contacted, authentication is always failed. This failure should be observable on the user interface.
Setup
  • Cycle the controller power.
Actions
  1. Start the Android user interface.
  2. Press the open/close button.
  3. Verify that the status message says "Processing..."
  4. Verify that the status message says "Authentication Failure"
Expected The Android user interface display should display that authentication failed.
Results PASSED
Cleanup
Requirements GDO-PR-UI-020, GDO-PR-UI-020.001, GDO-PR-UI-020.001.004
