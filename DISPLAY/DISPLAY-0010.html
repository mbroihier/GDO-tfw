Test Case DISPLAY-0010
Title Initial Android App Display
Objective

This test case is intended to verify the following implied or specified requirements:

  • The GDO user interface shall have one button that will be used to command the door open or closed.
  • The GDO user interface shall display a momentary status message after the button is pressed that shall display the status of the command attempt.
  • The GDO user interface shall display a communication failure message if it receives no response from the Raspberry PI 0 W within five seconds.
  • The user interface shall be implemented on an Android Device that supports bluetooth.
  • When the Android app is started a main page shall display as a minimum:
  • A text field that displays the status:
  • The app shall display a status of "Ready" when it is started.
  • The app shall display a status of "Communication Failure" when it fails to send a command or receive a response from the controller.
  • Display a command button.

During this test case, the garage door opener app will be started and the user will confirm that the display is correct when the Android device is not connected to the relay controller.
Setup
  • If enabled, disable bluetooth on the Android device.
Actions
  1. Start the garage door opener Android app.
  2. Verify that there is a status message display saying the app is ready to send a message.
  3. Verify that there is one button that says it is for opening or closing (the door).
  4. Press the open/close button.
  5. Verify that the app displays a message saying that communication failed.
Expected The app, when started, displays a ready status. When the open/close button is pressed the app presents a "processing" message, and when the app receives no reply form the relay controller, the app presents a message indicating communication failure.
Results PASSED
Cleanup
