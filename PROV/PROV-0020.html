Test Case PROV-0020
Title Controller is not automatically provisioned
Objective

This test case is intended to verify the following implied or specified requirements:

  • The bluetooth connection shall be manually provisioned via code in the controller program and/or Linux OS.

To improve security, this code requires hand configuration. That is, it requires code modifications requiring the user to compile in some configuration information and hand configure other parts of the configuration. This test focuses on the controller.
Setup
  • If the controller is paired with the UI, instruct it to forget the pairing.
Actions
  1. Inspect the controller code and verify that there is an "advertise_service" command that advertises a UUID that matches the UUID in the Android UI app.
  2. Pair the controller with the app:
    • sudo bluetoothctl \bullet
    • discoverable on \bullet
    • agent on \bullet
    • pairable on \bullet
    • on Android device, turn bluetooth on and pair with the node the controller is running on \bullet
    • turn off discovery, pairing, and agent on the controller. \bullet
    • <control> d to exit bluetoothctl \bullet\step
  3. Start UI app on the android device.
  4. Press the open close button.
  5. Verify that there was no communication failure.
Expected Provisioning is hardcoded in the controller and bluetooth pairing must be done using Linux bluetooth commands.
Results PASSED
Cleanup
