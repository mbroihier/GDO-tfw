Test Case PROV-0010
Title Android UI is not automatically provisioned
Objective

This test case is intended to verify the following implied or specified requirements:

  • The bluetooth connection shall be manually provisioned via Java code in the Android App and/or Android OS.

To improve security, this code requires hand configuration. That is, it requires code modifications requiring the user to compile in some configuration information and hand configure other parts of the configuration. This test focuses on the user interface.
Setup
  • If the Android device is paired with the controller, instruct it to forget the pairing.
Actions
  1. Inspect the UI code and verify that sets the service ID to the same service ID that is set in the controller code and verify that the UI code looks for the node name given to the controller.
  2. On the controller, enable bluetooth discovery and start the agent.
    • sudo bluetoothctl \bullet
    • discoverable on \bullet
    • agent on \bullet
    • pairable on \bullet
    • On the Android device, turn bluetooth on and pair with the controller node. \bullet
    • Turn off discovery, pairing, and agent on the controller. \bullet
    • Exit bluetoothctl with <control> d \bullet\step
  3. Start the app on the Android device.
  4. Press the open close button and confirm that no communication error occurs
Expected Provisioning is hardcoded in the UI and bluetooth pairing must be done using Android configuration routines.
Results PASSED
Cleanup
